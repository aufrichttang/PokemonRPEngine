# Whitepaper -> Runtime Mapping

This project maps four whitepapers into executable runtime rules.

## 01_worldview_whitepaper_v2
- Maps to `lore kernel`:
  - balance/dependency/instability
  - protocol phase progression
  - cross-signature growth

## 02_time_cosmology_whitepaper_v1
- Maps to `time kernel`:
  - temporal debt
  - narrative cohesion
  - judicative stability
  - compilation risk
- Adds event class system:
  - fixed / fragile / unjudged / echo

## 03_human_factions_and_official_legacy_v1
- Maps to `faction kernel`:
  - league / white_ring / consortium / grassroots / witnesses
- Adds canon legacy tags:
  - rocket / galactic / plasma / flare / aether / skull / rr archive

## 04_codex_prompt_lore_kernel_integration
- Mapped as implementation constraints:
  - additive APIs
  - deterministic backend mutation
  - memory metadata + retrieval filter
  - testing coverage for migration/state transition
